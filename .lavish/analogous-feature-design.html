RFC hex-palette v0.2.0

新功能：Analogous 色板生成

在现有 complementary 基础上，新增「近似色」(analogous) 调色方案 — 色轮上相邻的颜色组合，视觉上更和谐。

🤔 问题

当前 generatePalette() 只生成 complementary 方案。对于需要柔和、和谐配色的场景（如品牌设计、渐变背景），complementary 的强对比不够用。

用户需要 analogous（近似色）方案：色轮上相邻 ±30° 的颜色，视觉上自然协调。

💡 提议的 API 

import { generateAnalogousPalette } from './src/index.js';
const palette = generateAnalogousPalette('#3498db');
// {
//   base:    '#3498db',
//   warm:    '#5b34db',  // +30° 色轮
//   cool:    '#34db8a',  // -30° 色轮
//   light:   '#71b7e6',
//   dark:    '#246a99',
// }

新增函数签名

函数参数说明
rgbToHsl() {r,g,b} RGB → HSL 转换（内部辅助）
hslToRgb() {h,s,l} HSL → RGB 转换（内部辅助）
rotateHue() hex, degrees 在色轮上旋转指定角度
generateAnalogousPalette() hex, angle? 生成近似色调板（默认 ±30°）
